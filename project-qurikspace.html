Project Idea

QurikSpace focuses on a practical campus problem: students often have skills they can offer, but they need a focused place to list services, receive work requests, coordinate with clients, and build credibility through reviews.

Core Workflow

  • Service PublishingStudents create and manage gigs so their skills are visible as structured offerings.
  • Order ManagementClients can place requests while students track work and delivery status.
  • CommunicationBuilt-in messaging supports project discussion between clients and student freelancers.
  • PaymentsThe platform includes payment handling as part of the service-order flow.
  • Trust LayerReviews and personalized recommendations help users find better matches over time.

Deployment

The application is containerized with Docker, supported by GitHub Actions for CI/CD, and deployed publicly on Render so it can be tested as a real web application instead of only a local prototype.

Outcome

  • Delivered a live full-stack platform with marketplace, communication, payment, review, and recommendation features.
  • Designed around student freelancers, making the project more domain-specific than a generic gig listing app.
  • Added deployment and automation practices through Docker, GitHub Actions, and Render hosting.
