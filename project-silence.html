Creative Idea

The project is built around a very human question: when someone delays replying, what could that silence mean? Instead of treating the answer as a single guess, the app presents a ranked probability view so the user can see uncertainty and compare possible intent classes.

How It Works

  • Message CleaningNormalizes the text by trimming and compressing spacing before feature extraction.
  • Text VectorizationUses a saved TF-IDF vectorizer to convert the message into machine-readable text features.
  • Delay FeatureAdds reply delay as a normalized signal, capped for stability across short and very long delays.
  • Extra SignalsChecks punctuation, urgency words, negative phrases, and message length as additional behavioral cues.
  • PredictionLoads a trained classifier with Joblib and displays the top intent, confidence, runner-up, and full probability ranking.

User Experience

The Streamlit interface is designed like a small prediction workspace. Users can select examples, write their own message, choose delay in minutes or hours, and instantly review confidence, reliability notes, probability bars, and a graph view.

Outcome

  • Created an end-to-end NLP app with model artifacts, UI, prediction logic, and deployment-ready requirements.
  • Balanced machine learning output with human-readable explanations so the result feels understandable instead of mysterious.
  • Added fallback intent scoring logic, making the app more resilient when model labels do not match the expected intent classes.
